46.PageInherit

수정 일자 버전 (SP2)

  • WebSquare5 Studio : 20180117_1349_x86_B

  • WebSquare5 Engine : 5.0_2.2982B.20180223.154326_1.5

일반적인 Include처럼 다른 웹스퀘어 xml화면 파일을 현재 화면에 include하고자 할때 사용하는 컴포넌트입니다. 가장먼저 로드되기때문에 공통적인 모듈이나 개발단을 묶어서 include 할때 사용하기 좋은 컴포넌트 입니다.

UI 공통을 작성하고, dataCollection, instance, JavaScript, CSS에 대한 상속 기능을 제공하는 컴포넌트 입니다.

46.1Property

Property

Description

사용자가 임의의 class를 정의할때 사용합니다.

userData1

46.3주의사항 및 Tip

  • 주로 공통이 되는 top, left 메뉴 등에서 사용합니다. 동적으로 setSrc를 하며 화면을 바꿔야 할때는 iframe을 사용해야 합니다. (pageInherit은 include시 한번만 src설정할수있음)

46.4WFrame과의 차이점

  • dataCollection, instance, JavaScript, CSS를 공통화 하는 기능을 wframe에서 제공하지만 wframe을 포함한 페이지에서 wframe에 선언된 dataCollection, instance, JavaScript, CSS를 재정의(overriding)할 수 없다는 문제점이 있습니다.

  • pageInherit 컴포넌트는 wframe의 문제점을 극복할 수 있는 컴포넌트 입니다.

  • 새로 삽입하는 웹 페이지의 DataCollection, 인스턴스, JavaScript, CSS를 재정의할 수 있습니다.
