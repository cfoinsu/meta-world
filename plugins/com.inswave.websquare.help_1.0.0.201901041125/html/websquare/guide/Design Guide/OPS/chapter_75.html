66.Tag

웹스퀘어5에서는 html을 직접 사용이 가능합니다. 따라서, tag 컴포넌트를 혼용해도 무방합니다.

Tag 컴포넌트는 다른 컴포넌트와 달리 엔진에서 데이터를 바인딩, API 제공 없이 사용자가 지정한 tagName을 그대로 by pass 해주는 방식(자바스크립트 적으로 설명하면 documnet.write, innerHTML)입니다. 그렇기 때문에 데이터가 바인딩되지 못하기 때문에 개발자가 파싱된 컴포넌트에 대해 핸들링이 되지 않아 사용시 개발자 혹은 기술지원 담당자와 상의해야 합니다.

데이터 바인딩을 해야할 경우에는 웹스퀘어5에서 제공하는 컴포넌트를 혼용해서 사용하며 데이터 바인딩이 가능한 컴포넌트를 사용해야 합니다.

66.3속성 추가하기

Tag 컴포넌트가 자주 쓰이는 Table코딩 심화의 예제를 들어 설명하겠습니다.

Tag 컴포넌트로 웹표준 및 접근성을 고려한 Table 코딩을 하기위해 table의 summary, colspan, rowspan등의 속성을 필수적으로 삽입해야 하는 경우가 있습니다. 그를 위해서Tag 컴포넌트에 Group이나 Textbox와 같이 Attribute를 추가하려고 오른쪽 클릭으로 컨텍스트 메뉴를 호출해 보았는데 컨텍스트 메뉴에 '속성 추가하기'가 존재하지 않습니다.

앞장 Table 코딩 심화에서 말한 것과 같이 Tag 컴포넌트는 by pass해 주기때문에 Tag 컴포넌트는 Source 편집기로 이동하여 직접 Attribute를 입력해도 그대로 파싱됩니다. Tag 컴포넌트의 Attribute를 입력하시려면 Source 편집기로 이동하여 직접 입력합니다.

66.4form 코딩

Websquare은 기본으로 ajax 통신을 기본으로 하고 있어 데이터 통신에는 form 요소를 별도로 코딩하지 않습니다. 그렇기에 사용이 필요할 시에는 담당 개발자와 담당 기술지원 인력에게 가이드 받으시길 바랍니다.

그래도 form 코딩을 해야된다고 결정날 경우에는 Form 태그는 화면에 보여져야 할 경우가 적기에 Source 편집기에서 직접 코딩하여도 상관이 없습니다. 하지만 웹 표준 및 접근성과 같은 이유로 form 태그 안에 hidden이 아닌 InputBox와 같은 폼 요소의 컴포넌트를 사용해야 할 시에는 Tag 컴포넌트를 사용하여 Design View에 반영이 되게끔 코딩합니다. Tag 컴포넌트를 사용하지 않고 코딩을 하게 되면 소스를 직접 보지 않고서는 Design View와 Component View에 반영이 되지 않아 유지보수에 어려움이 있을 수 있습니다.

66.6주의사항 및 Tip

  • Tag 컴포넌트는 by pass해 주기때문에 Source 편집기로 이동하여 직접 Attribute를 입력해도 그대로 파싱됩니다. Tag 컴포넌트의 Attribute를 입력하시려면 Source 편집기로 이동하여 직접 입력합니다.

  • 데이터가 바인딩되지 못하기 때문에 개발자가 파싱된 컴포넌트에 대해 핸들링이 되지 않아, 사용시 개발자 혹은 기술지원 담당자와 상의해야 합니다.
