[default:false, true] spa 모드 사용 시 빈iframe을 createWindow가 일어난 후에 1개씩만 생성하도록 하는 속성. 이 속성은 spaInitCount보다 우선순위가 높다. 즉 이 속성이 true이면 spaInitCount 설정은 무시된다. 창이 최대로 생성된 경우에는 빈 iframe을 생성하지 않는다. (버전 정보: 5.0_2.2777B.20170904.121217 버전에서 추가)

spaAutoDelay SP2

[default:false, true] spaAuto 속성 사용 시 createWindow를 수행한 후 몇 ms 지난 후에 빈 iframe을 생성할지를 지정하는 속성. 이 속성값보다 적은 간격으로 createWindow가 연속적으로 호출된 경우, 마지막 createWindow가 호출된 후에 빈 iframe을 1개 생성한다. 기본값은 3000ms. 즉 createWindow가 호출되고 3초 뒤에 빈 iframe을 생성한다. (버전 정보: 5.0_2.2777B.20170904.121217 버전에서 추가)

spaInitCount

windowContainer생성 시 spa용 iframe을 미리 생성해 둘 개수를 지정합니다.. iframe을 미리 생성하여 createWindow 속도를 향상킬수 있습니다. 0보다 큰 값일 경우 spa모드로 동작하며 0이 아닌경우 일반적으로 windowMaxNum과 같은 값을 설정합니다.

spaReplaceHistory SP2

SPA 사용 시 Window 영역(IFrame 내부)의 history(앞으로 가기 및 뒤로 가기) 동작을 금지. IFrame 내부 페이지의 뒤로 가기 동작을 막아서 blank 페이지로 이동하지 않는 효과. 이 경우, 뒤로 가기를 수행 할 경우 브라우저 전체가 뒤로 이동. true: 뒤로 가기 및 앞으로 가기 동작을 금지. false (기본 값): 동작을 허용.

75.2windowContainer 적용사례

  1. Treeview를 이용한 windowContainer 탭 생성

    • treeview를 더블클릭하여 Inner XML모드에서 Value값에 URL을 넣습니다.

      (예: /root/aaa.xml)

    • treeview컴포넌트 마우스 우측에 onlabelclick 이벤트를 선택합니다.

      (예: scwin.treeviewID_onlabelclick = function(value, nodeElement, index){})

    • treeview의 클릭이벤트에 받아온 value 등 인자를 조건으로 windowContainer의 탭을 생성합니다.

      (예: if(value.length > 0){windowContainerID.createWindow(this.getSelectedLabel(), null, value);}

      샘플: use_treeview_create_windowcontainer.xml

  2. anchor를 이용한 windowContainer 탭 생성

    • anchor컴포넌트 마우스 우측에 onclick 이벤트를 선택합니다.

      (예: scwin.anchorID_onclick = function(){})

    • 위의 함수에 다음과 같은 api를 호출하여 생성합니다.

      (예: windowContainerID.createWindow("메인", null ,"/root/aaa.xml"))

      샘플: use_normalLink_create_windowcontainer.xml

75.5주의사항 및 Tip

  • 보통 레이아웃용으로 쓰입니다.
