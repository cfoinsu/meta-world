53.ScheduleCalendar

수정 일자 버전 (SP2)

  • WebSquare5 Studio : 20180117_1349_x86_B

  • WebSquare5 Engine : 5.0_2.2982B.20180223.154326_1.5

ScheduleCalendar는 'FullCalendar 외부 라이브러리'를 연동하여 일정을 calendar에 출력하는 컴포넌트입니다.

WebSquare5에서는 이를 Customizing 할 수 있도록 API를 지원합니다.

FullCalendar 에 대한 자세한 Properties & API는 아래의 공식 홈페이지를 참조하시기 바랍니다.https://fullcalendar.io/

53.3class 적용과 스타일 변경

[그림 53-2]ScheduleCalendar default 스타일

웹스퀘어에서는 xml파일의 ScheduleCalendar 컴포넌트를 html로 랜더링 되는 과정에 엔진에서 w2scheduleCalendar 라는 class를 파싱합니다,(w2로 시작하는 클래스는 웹스퀘어 엔진에서 사전에 정의되어 있습니다.)

ScheduleCalendar 컴포넌트는 FullCalendar 외부 라이브러리와 연동되어 있기 때문에, fc 라는 class 로도 파싱됩니다. 컴포넌트를 구성하는 하위 요소들 또한 fc 로 시작하는 class로 정의되어 있습니다.

위에 예제는 ScheduleCalendar 컴포넌트의 Property속성 중 class명을 schedule_sample으로 주었습니다. html 로 랜더링 시 w2scheduleCalendar 바로 뒤에 연결되며 후 순위로 적용됩니다.

여기서 css 적용 우선순위를 주의하셔야 합니다.

기본적으로 웹스퀘어의 공통 외부 스타일시트 파일인 stylesheet_ext.css 파일이 가장 먼저 로딩됩니다. 이후 사용자는 인라인 방식이나 외부 스타일시트 파일을 참조하여 기본 class를 overriding할 수 있었습니다.

ScheduleCalendar에서는 stylesheet_ext.css -> 사용자 정의 외부 스타일시트 -> (마지막) fullcalendar.css 와 같은 순서로 로딩됩니다. FullCalendar의 공통 css 후에 사용자 정의 외부 스타일시트가 로딩되어야 할 경우 Property 창에서 cssRenderOrder 속성을 true 로 설정하시면 됩니다. (해당 속성은 5.0_2.2982B.20180223.154326_1.5 부터 적용됨)

해당 속성을 이용하지 않을 경우에는 !important 를 이용하여 우선순위를 조정하시면 됩니다.
