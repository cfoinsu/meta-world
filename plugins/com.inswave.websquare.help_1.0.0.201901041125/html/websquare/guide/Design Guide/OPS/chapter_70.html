61.Span

수정 일자 버전 (SP2)

  • WebSquare5 Studio : 20180117_1349_x86_B

  • WebSquare5 Engine : 5.0_2.2982B.20180223.154326_1.5

Text를 출력하는 컴포넌트로 HTML span Tag과 동일한 기능을 제공하며 브라우저에는 span Tag로 표현 되므로 기본 display타입은 "display:inline" 으로 표현됩니다.

Textbox의 tagname속성을 이용하여 span Tag를 표현하던 번거로움을 개선하기위해 생성된 컴포넌트입나다.

Textbox와 비슷한 기능을 가지고 있으며 tagname속성은 포함되어있지 않습니다.

61.1Property

Property

Description

class

HTML의 class속성과 동일한 기능을 제공하며 css파일 또는 style블럭에 정의 한 class를 1개 또는 공백(space)를 이용하여 다중 적용이 가능하다.

dataType

[default:text, number, date, time, link]컴포넌트 value의 data type을 명시하며 해당 값은 엔진에서 제공하는 displayFormat등의 속성에서 참조한다.

displayFormat

엔진 내부에서 dataType 속성에 정의한 type에 따라 동작되며 excel의 서식을 적용하는 방식처럼 value의 format을 적용 할 수 있다.displayFormatter와 동시적용이 불가하다. 예시)dateType이 number일 경우 #,###.#0 으로 적용하면 3자리수 마다 ,가 찍히며 소수점 2자리까지 표현된다.

id

컴포넌트의 ID로 전역객체로 할당되며 script에서 본 id로 컴포넌트에 접근이 가능하다.

label

컴포넌트의 value로 화면에 출력하고자 하는 text를 정의한다.

style

InputBox의 style 속성

tabIndex SP2

Tab Key를 이용 한 컴포넌트의 포커스 순서로 HTML의 tabindex속성과 동일한 기능을 제공한다.

html attributes

해당 테그에 대응하는 속성을 추가한다.

61.2"span" 컴포넌트의 브라우저 표현

[그림 61-1]01

해당 캡쳐 화면과 같이 span컴포넌트를 추가할경우 브라우저에 표현돼는 tag는 "span" tag로 표현이 된다. 그렇기때문에 "span" tag의 기본display타입이 "display:inline" 이므로 해당화면과 같이 정렬이 됩니다.

61.4주의사항 및 Tip

  • 본 컴포넌트는 Textbox의 tagname속성을 span으로 표현하는 번거로움을 개선하기위해 만들어진 컴포넌트이기 때문에 tagname과 같은 속성은 없으며 컴포넌트를 생성하면 span테그로 파싱 되어진다.
