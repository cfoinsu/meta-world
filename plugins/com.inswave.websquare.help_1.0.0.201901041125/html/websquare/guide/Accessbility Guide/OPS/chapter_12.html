10.1GridView

그리드뷰 컴포넌트의 접근성 설정은 컴포넌트 자체에 설정하는 “컴포넌트 설정”과 각 컬럼의 inputType에 따른 “컬럼 설정”으로 구분되고, 그리드의 성격에 따라 readOnly 속성을 true(조회용), false(편집용) 값으로 설정하여 사용합니다. 이때 부분적으로 편집이 가능한 컬럼이 필요한 경우 각 컬럼에서도 readOnly 설정을 할 수 있습니다. 예) 조회용 그리드 이지만 데이터 선택을 위해 첫 컬럼이 radio나 checkbox로 되어있는 경우 컴포넌트 설정은 readOnly=”true”로 하고 해당 컬럼의 readOnly=”false”로 설정.

10.1.1컴포넌트 설정

senseReader=”true” focusMove=”false” keyMoveEditMode=”true” autoResizeEmbeddedInput=”true”로 설정하면 그리드 컬럼에 컴포넌트들은 embedded 형태로 랜더링 되어 keyboard tab키 만을 이용하는 환경이나 센스리더와 같은 음성 낭독 프로그램을 사용하는 환경에서도 제약 없이 이용할 수 있습니다. 또 다음과 같은 접근성 관련 속성을 이용하면 보다 웹 접근성이 보장되는 그리드를 제작할 수 있습니다.

10.1.2ID/Headers 자동 생성 방법

복잡한 형태의 그리드 테이블의 경우 th(제목)의 ID와 td(내용)의 Headers 속성을 이용하여 제목과 내용을 연결하기 위하여 ID매칭 작업을 완성 한 후 “TD Headers 생성” * Headers 생성 후 그리드를 수정해야 하는 경우 수정이 모두 끝난 후 “TD Headers 생성”을 통해 새롭게 갱신하여야 한다.

10.1.3Tab키 이동시 그리드 시작과 끝에서 포커스가 사라지는 현상

이 현상은 그리드내 가상스크롤을 사용하면서 데이터의 제일 처음과 끝으로 Tab키를 이동을 하고 다시 Shift+Tab으로 이동을 하면 데이터의 처음과 끝을 알수가 없기 때문에 div를 추가하여 데이터 처음과 끝으로 이동 시켜준다. Tab이동이 되면 센서리드에서 텍스트를 읽어주어 지금 어떤 상태라는걸 알 수 있다.
