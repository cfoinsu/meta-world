3.4.2. Proxy 적용

Dynamic Web Project를 WAS (web Application Server)를 통해 실행할 때, WebSquare XML Page도 실행할 수 있습니다. 단지 WebSquare XML은 WebSquare Engine을 통해 해석 되어야 하기 때문에 WebSquare Engine이 Proxy Server의 역할을 할 수 있도록 Setting이 필요합니다.

다음의 예는 WebSquare Sutdio에서 Dynamic Web Project를 생성하고 WebSquare XML 에디터의 preview를 통해 WebSquare XML이 browser를 동작함을 보인 것입니다.

WebSquare Studio의 Menu-bar에서 File à New à Other… 를 선택합니다. New 창이 활성화 되면 다음과 같이 Web à Dynamic Web Project 를 선택합니다.

 

 

Java Source가 저장될 path 지정, Context root 지정, Content Directory 지정의 과정에서 원하는 값을 지정하고 Next 버튼을 눌러 Project name을 기입하고 Next 버튼을 눌러 다음 과정으로 넘어 갑니다. 후 Finish를 눌러 Dynamic Web Project를 생성 완료 합니다. (예제에서는 default 값 그대로 생성하였습니다.) 그러면 다음과 같이 Project Explorer에 생성된 Dynamic Web Project를 확인할 수 있습니다.

 

프로젝트의 설정이 “WebContent” 이하에 Web Page 들이 저장되도록 되어 있기 때문에 이 폴더에 WebSquare XML 파일을 하나 생성합니다. WebSquare XML 파일 생성은 Project Explorer의 Dynamic Web Project 하위의 “WebContent” 폴더를 선택하고 마우스 오른쪽 버튼을 사용하여 컨택스트 매뉴를 활성화 하여 New à Other…를 선택합니다. New 창이 활성화 되면 WebSquare à WebSquare XML을 선택하고 Next 버튼을 누르고 생성될 파일명을 기입한 후 Finish 버튼을 눌러 생성합니다.

 

 

상단의 화면 캡쳐와 같이 WebSquare XML 파일이 생성되고 WebSquare XML 에디터가 활성화 됨을 확인 할 수 있습니다. 여기에 다음과 같이 calendar 컴포넌트를 하나 삽입합니다. WebSquare Engine이 Proxy Server의 역할을 수행해 컴포넌트가 제대로 표시되는지를 확인하기 위함입니다.

 

이 때, Project Explorer의 WebSquare XML 파일의 아이콘을 보면 상단에 빨강색 원의 표시가 있습니다. 이는 WAS가 실행되지 않아 현재 WebSquare XML이 Preview Tab에서 실행 될 수 없음을 표시합니다. WAS를 실행하면 빨강색 원의 표시가 녹색의 원의 표시로 변경 됩니다. 지금 WAS를 실행합니다.

WebSquare XML이 WebSquare Engine으로 해석되기 위해서는 WebSquare Engine을 Proxy로 설정해야 합니다. 설정을 위해서는 WebSquare Preview Setting이 되어 있어야 합니다. Preview Setting의 생성은 2.1절, 2.2절을 참조하시기 바랍니다. Project Explorer에서 Dynamic Web Project를 선택하고 마우스 오른쪽 버튼으로 컨택스트 매뉴를 활성화 합니다.

 

 

화면 캡쳐와 같이 컨텍스트 매뉴에서 Configure à Add WebSquare Support 를 선택합니다. 그러면 다음과 같은 WebSquare Preview 창이 활성화 됩니다. 예제에서는  WebSquare Studio의 Embedded Engine을 proxy로 설정하였습니다.

 

Finish 버튼을 눌러 Proxy 설정을 완료 합니다.

이제 WebSquare XML 에디터의 Preview Tab에서 WebSquare XML의 컴포넌트가 제대로 표시됨을 확인합니다.
